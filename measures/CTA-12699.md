## 施策背景

- Googleは200以上の指標で記事の順位付けをしており、近年エンゲージメント指標（直帰率・記事内滞在時間・UU・CVR）を特に重要視している
- CTはHTと比べて直帰率が約10%高く、10秒以上滞在比率が約10%低い
- 現在のCT記事でユーザーが知りたい情報へのアクセスを困難にさせている課題:

**FV全体の課題:**
- 各コンポーネントが識別しにくい
- 各要素の表示が大きく、表示される情報量が少ない
- H2・H3・H4が区別しにくい

**各コンポーネントの個別課題:**
| コンポーネント | 課題 |
|---|---|
| ヘッダー | 縦幅が大きく本文の表示画面を狭めている |
| パンくず | 現在いる階層がわからない（関連記事へのアクセスがしにくい） |
| タグ | 記事最下部にあり＆セクションが小さく、ユーザーが気付けない |
| 最終更新日 | 文字が薄くて認識しにくい |
| タイトル | 文字サイズが大きくて読みにくい |
| このページのまとめ | 文字の羅列でセクションになっておらず、「まとめ」と認識しにくい |
| 目次 | H2とH3が区別しにくく、開きっぱなし状態で本文への距離が遠い |
| 見出し | H2とH3の区別がしにくい（行間・見出しデザイン） |
| 囲み | 黒罫線で切り離されているように見え、コンテンツ内容を認識しにくい |
| FAQ | 見出しと同じ配色・スタイルで、FAQセクションと認識しにくい |
| CAのアドバイス | 見出しと同じ配色・スタイルで、CAアドバイスと認識しにくい |
| フッター追従バナー | FVから表示されるため、タイトル〜目次の画面で本文の表示領域を狭めている |

## 仮説（施策ドックより・事実）

上記の課題を解決することにより、情報が整理され、ユーザーのエンゲージメント指標向上に寄与できる可能性がある。

## 事実（PDFより確認済み）

- ABテストではなく前後比較のため、季節性バイアスが含まれる（セッション数が両デバイスで約30%減少）
- SPはエンゲージメント時間+9%、10秒以上滞在率+1.2pt、直帰率-1.2pt と全体的に改善傾向
- PCはエンゲージメント時間が-5.4%と低下。スクロール到達率は微増、直帰率は横ばい
- 有意差なし（前後比較の性質上、統計的判断は難しい）
- ph1として実施。ph2への継続が示唆されている

## AI推論（仮説・未確認）

> ⚠️ 以下はデータから推論した仮説であり、事実確認はされていない

- まとめボックス・目次の視認性向上がページ内滞在の質を上げた可能性がある（エンゲージメント時間+9%）
- SPのセッション数が約30%減少しており、前後比較の季節性バイアスが結果に影響している可能性がある
- PCユーザーはもともとエンゲージメントが高く、UIの追加改善が効果として表れにくい状態だった可能性がある

## 要因分析（根本原因）

SPではエンゲージメント時間の改善が確認でき、UI整理の方向性は正しい。ただし直帰率（KPI）の改善が-1.2ptにとどまった。FVのUI改善だけでは「記事に入ってすぐ離脱する」行動は変えられず、タイトル・冒頭文と検索意図のマッチング精度を別途改善する必要がある。

## 学び / 気づき（施策ドックより）

**UIの改善はSPでより効果が出やすい**
- PCはもともとエンゲージメントが高くUI改善の追加効果が出にくい
- SPは複数指標で改善が見られ、モバイル体験の改善が読了行動に直結しやすい
- SEO記事のUI改善はSPを主軸に設計・検証するのが効率的

**まとめボックス・目次はスクロールを促す効果がある**
- 記事冒頭で「何が書いてあるか」を伝えることが読み進める動機づけになる

**直帰率の改善には別のアプローチが必要**
- 読み始めたユーザーの質は上がったが、離脱するユーザーの行動は変わっていない
- 直帰率を下げるにはUI改善だけでなく、検索意図とコンテンツのマッチング精度（タイトル・冒頭文）の見直しが必要
